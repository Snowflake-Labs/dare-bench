This is a dataset that is generated by stimulating common fraud patterns, and is intended to simulate some of the nuisances when dealing with fraud detection.

Common fraud detection patterns used in the dataset includes:
- unusual transactions
- identity theft
- high frequency transactions
- "testing water" (unusual small transactions followed by large transaction).

This is further complicated by stimulating both common and uncommon non-fraud purchase patterns in clients.
Both fraud and legitimate transactions can have dependencies with each other.

=== About this file ===

About this file
Data simulates 1 year of transactions for 3,000 clients.


=== Columns & descriptions ===

client_id: unique ID of client
transaction_id: Unique ID of transaction
timestamp: Time stamp of transaction in system time
transaction_amount: amount in transaction
merchant_type: type of merchant transaction is for
transaction_type: What system is used to execute transaction
currency: currency of the transaction
city: Canadian City where transaction occurred
email_address: email address of the client on file at the time of transaction
Fraud_Label: Binary Label for whether the transaction is fraudulent or not
credit_limit: [no description]
